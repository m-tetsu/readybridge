# RAG 巡回ソース台帳

> RAG（AI相談）の収集パイプラインが巡回する公的ソースの管理台帳。
> 設計の背景・方針は `docs/rag-plan.md`（論点③）を参照。
> このファイルはソース追加・変更のたびに更新する。
> 最終更新：2026-06-19 ／ ステータス：**雛形（URL・robots・規約は要精査）**

## 凡例

- **層**：A=恒常的な支援の土台 / B=災害ごとの特設 / C=自治体（当面スコープ外）
- **robots**：`/robots.txt` で対象パスが許可か（要確認＝未チェック）
- **更新日取得**：ページから「最終更新日」を機械的に拾えるか（鮮度表示に使う）
- **規約**：再利用条件のメモ（多くは政府標準利用規約相当だが個別確認が必要）
- **巡回**：✅=有効 / ⏸=枠のみ（災害時に追加） / ⛔=対象外

## クローラのスコープ設計

クローラ（`tools/crawler/`）は**シードURLを起点に、許可ドメイン内をリンクで辿る**。機械可読な設定の正は `tools/crawler/crawl-config.json`（この台帳は人間向けの一覧）。

- **許可ドメイン**：`.go.jp`（国）/ `.lg.jp`（自治体）。シードが他省庁・県ページにリンクしていれば、それらも辿って取り込む。
- **安全弁**：最大深さ（既定2）＋1回あたり最大ページ数（既定300）＋リクエスト間隔（礼儀）。go.jp/lg.jp 全体への無制限クロールにはならない。
- **pathPrefix**（任意）：シードと同一ホスト内のみパスで絞れる（恒常ソース向け）。越境リンクには適用しない。
- シードの `pillar`/`category`/`disaster_scope` を、そこから辿ったページが引き継ぐ。

## C層（自治体）— 当面スコープ外

数が多く構造もバラバラなため後フェーズ。対象化する際はこの台帳に追記する（巡回＝⛔のまま保留）。

## 運用ルール（要約）

- robots.txt を尊重し、Disallow 対象は巡回しない。
- 低頻度（日次1回程度）・User-Agent 明示・リクエスト間隔を空ける。
- 利用規約・再利用条件はソースごとに個別確認。回答では必ず出典（URL・更新日）を明示し、全文転載はしない。
- 取得失敗・構造変化を検知したら取り込まず**スキップ＋通知**。差分があった時だけ PR 起票し人がレビュー。
